Technical Specification

OPD Claim Adjudication System

An automated, AI-powered system designed to process medical receipts and adjudicate Outpatient Department insurance claims against policy terms.

01 System Architecture

The application is designed using a decoupled client-server architecture with stateful document storage and external AI service integrations.

02 API Specifications

The backend API exposes RESTful endpoints for submitting claims, fetching audit lists, appealing, and managing review states.

POST /api/claims

Submit a new claim containing medical documents (prescriptions/bills) for instant automated adjudication.

Form-Data Body

  • member_id (string): Unique employee ID (e.g., EMP001)
  • member_name (string): Full name of the patient
  • treatment_date (string, YYYY-MM-DD): Date of treatment
  • claim_amount (number): Total claim value in INR
  • documents (files): Images/PDF files of bills & prescriptions

Response Example

{
  "success": true,
  "claim_id": "CLM_782A9",
  "decision": "APPROVED",
  "approved_amount": 1350,
  "rejection_reasons": [],
  "confidence_score": 0.96,
  "notes": "Consultation covered. 10% co-payment applied.",
  "document_urls": ["https://res.cloudinary.com/.../bill.jpg"]
}
POST /api/claims/:id/adjudicate

Allows an administrator to manually review, override, and force approve/reject a claim.

JSON Body

{
  "decision": "APPROVED",
  "approved_amount": 4500,
  "notes": "Override: Approved upon visual proof review of invoice."
}
POST /api/claims/:id/appeal

Allows a member to appeal a rejected or partially approved claim with updated justifications.

JSON Body

{
  "reason": "Doctor registration details are legible on page 2."
}

04 Key Policy Assumptions

The adjudication engine uses the following operational guidelines for processing claims:

  • Active Member Profiles: All claimants are assumed to be registered in the system database with active policy definitions.
  • Consultation Fee Co-Pay: A 10% co-payment is strictly applied only to doctor consultation fees. Medications, diagnostic lab tests, and procedures are covered at 100% (unless excluded).
  • Per-Claim Limit: Any single claim submission is subject to a maximum cap of ₹5,000. Claims submitted above this value are rejected automatically with the reason code `PER_CLAIM_EXCEEDED` (or partially paid out up to the limit).
  • Minimum Claim Payout: Claims with total billed values under ₹500 are automatically rejected to prevent high processing overhead for low-value transactions.
  • Treatment Window: Claims must be submitted within 30 days of the treatment date. Backdated claims exceeding this period are rejected for late submission.
